## Verification
- `docs/world_class_gap_analysis_and_execution_plan.md` now contains the tabular gap analysis, phased implementation plan, checkpoint model, CPU mitigation plan, and adversarial analysis.
- `StrategyConsole` now offers a one-click handoff into a prefilled Forecast Registry draft using the structured multi-agent forecast output.
- `StrategyConsole` now also surfaces linked analysis review state and allows the analysis owner to request human review from the multi-agent forecast panel.
- `StrategyConsole` now reuses the same forecast governance helper as `ForecastRegistry`, showing a pre-handoff publish-governance preview and analysis-age freshness label before opening the registry draft.
- `ForecastRegistry` now consumes draft state/session storage, persists `analysis_run_id` and `game_theory_model`, trims list-query payloads, avoids a full list refresh after every prediction update, shows weighted-consensus detail metrics, surfaces trust labels, and blocks publish when draft scoreability checks fail.
- `ForecastRegistry` now hydrates linked `analysis_runs` review state, reflects human-review/evidence signals in trust labels, surfaces review status in draft/detail views, and blocks creation for drafts linked to rejected analyses.
- `ForecastRegistry` now derives a centralized publish-governance assessment from draft readiness, linked review state, disagreement metadata, and analysis age, separating hard blockers from review-required conditions and cautionary warnings.
- `ForecastRegistry` now fetches linked analysis `created_at` and exposes analysis-age freshness labels in both draft and detail views without introducing a forecast-level publish-state schema.
- `src/lib/forecastGovernance.ts` now provides the shared readiness, publish-governance, and analysis-freshness logic consumed by both Strategy Console and Forecast Registry to reduce policy drift.
- `src/lib/supabase.ts` now exposes the `brier-weighted-consensus` endpoint to the frontend.
- `src/lib/supabase.ts` now also exposes authenticated user-session headers for protected human-review endpoints.
- `src/lib/supabase.ts` now exposes the secured `forecast-create` endpoint for app-path forecast creation.
- `supabase/functions/brier-weighted-consensus/index.ts` is now read-safe by default and only mutates forecast aggregates when explicitly requested.
- `supabase/functions/forecast-create/index.ts` now revalidates publish-governance server-side for the app path, requires authentication, and prevents users from linking analysis runs they do not own.
- `supabase/functions/human-review/index.ts` now supports browser-safe CORS, reviewer-gated queue access, and an owner/reviewer `request-review` endpoint for analysis runs.
- `src/App.tsx` now exposes the existing `HumanReview` workflow through an admin-only `/reviews` route.
- `useStrategyAnalysis` now uses adaptive timeout-based polling instead of a fixed repeating interval.
- Workspace validation diagnosis showed that the primary immediate blocker was missing installed dependencies (`node_modules` absent). After installing dependencies and removing the incompatible `@types/react-router-dom` package for a `react-router-dom@6` app, both `tsconfig.app.json` and `tsconfig.node.json` now typecheck cleanly.
- The latest frontend validation passed with `pnpm exec tsc --noEmit --project tsconfig.app.json` after the shared governance refactor and secured forecast-create wiring.
