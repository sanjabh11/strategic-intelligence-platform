## Demo Opener

- **0:00-0:30 - Buyer workflow anchor:** Start with the buyer current workflow: what question is stuck, which decision depends on it, and what evidence they already collect.
- **0:30-1:15 - Evidence and uncertainty:** Show how the MVP gathers sources, marks uncertainty, and keeps claim boundaries visible instead of presenting a magic answer.
- **1:15-2:15 - Reasoning and review:** Walk through actor/countermove reasoning, forecast draft, human-review state, and what a reviewer can accept, reject, or send back.
- **2:15-3:00 - Measurable pilot ask:** Close on one pilot metric: time-to-brief, decision usefulness, objection quality, or pre-resolution forecast capture. State clearly that buyer validation and accuracy proof require the pilot evidence loop.

## Objection Handling

- **We already use risk intelligence tools.** Keep those tools. This pilot tests the workflow after information intake: turning one uncertainty into assumptions, decision options, reviewer notes, and a scoreable forecast draft.
- **Can you prove prediction accuracy?** Not prelaunch. The honest offer is evaluation readiness: pre-resolution capture, baselines, leakage review, and resolved-outcome scoring once real events resolve.
- **Is this secure enough for enterprise use?** Not for broad enterprise deployment claims yet. The prelaunch packet can show current security/AI-governance evidence and the exact controls still required before production use.
- **Can this integrate with our stack?** For MVP, avoid integration dependency. Run a guided pilot using exported evidence and a narrow workflow; integration becomes a later requirement after value is proven.
- **This sounds like consulting, not software.** The first pilot is founder-led to validate the workflow. The software value is repeatable evidence capture, review state, forecast governance, and reusable decision artifacts.
- **Who is the buyer?** Start with teams already paying for risk intelligence, foresight, advisory, treasury tooling, or executive training. The outreach sequence is designed to test which buyer owns the budget.
